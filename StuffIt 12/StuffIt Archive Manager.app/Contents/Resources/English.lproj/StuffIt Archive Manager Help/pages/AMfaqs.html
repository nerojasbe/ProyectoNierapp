Frequently Asked Questions

Q: Does StuffIt use "lossy" or "lossless" compression?

A: Lossy compression is commonly used in multimedia files (ie: audio, image and movie files). It's a process whereby information is "intelligently" discarded to create a smaller file. There is a sacrifice. Generally, the discarded information has a negligible affect on the file's image, sound or playback quality, but the process can be modified so that there is greater compression but lower quality. Because StuffIt can be used to store critical information such as spreadsheet or database files, it uses lossless compression, a process where files are made smaller without discarding any information. When a file is compressed and then expanded it is identical (bit for bit) to the original file - no data is lost.


Q:I am trying to compress files as small as possible. Any tips?

A: Some files compress better than others. Compression algorithms look for redundancy in files. Many multimedia files are already saved in a "lossy" compression format which reduces this redundancy. You may get some further compression on certain "lossy" compressed files (such as .gif images or .mp3 audio files) by using the SITx recompression options.

To do this go to DropStuff's preferences. In the StuffIt Preferences click on the Recompression tab. Check the boxes for each File Type that you are trying to compress.

Compressed files for which you may see little or no further compression include audio files such as MP3s; image files such as GIFs; movies such as MPEG and QuickTime .mov files; and others such as Adobe PDFs and Microsoft PowerPoint files. StuffIt always gets the most possible lossless compression possible out of every file.

Uncompressed multimedia formats such as .wav or .aiff (audio) and many high resolution graphics such as .psd (native PhotoShop picture), .eps (encapsulated postscript graphics), and uncompressed .tiff files will compress quite well. In fact, if you are starting out with a high resolution graphic you may get a smaller file with no loss of information by compressing into StuffIt X format instead of converting to a lossy format such as JPEG or GIF.

If you think your files are not of the already highly compressed varieties listed above, and you would like to get better compression, try adjusting the Custom Compression Settings to see if this improves your results.


Q: I have a password encrypted file for which I have forgotten the password. What can be done?

A: There is nothing that we can do in this case: the contents of your file has been encrypted based on the password you provided, it cannot be decrypted without this password. There is no "backdoor", and we are not aware of any weakness that would allow you to break the encryption, short of trying all possible passwords. We apologize if this has caused you to lose an important file, but encryption would be a worthless feature if it could easily be bypassed.

StuffIt Scheduler

Q: What happened to Archive Assistant?

A: The functionality of Archive Assistant has been incorporated into the StuffIt Archive Manager application.

Q: What happens if my computer isn't running when a task is scheduled to run?

A: If your computer is either shut down or in sleep mode when a task is scheduled to run, StuffIt Scheduler will be unable to launch it. The next time you start or wake up your computer StuffIt Scheduler will notify you that a scheduled task was not performed.



From this dialog you will have the chance to run the backup task immediately, or choose to skip that instance of the task and run it again when it is next scheduled.

Q: What CD and DVD burners are supported?

StuffIt Scheduler utilizes the disk burning software that is included with Mac OS X. If you CD or DVD burner is compatible with this software, it should work just fine with StuffIt Scheduler.
