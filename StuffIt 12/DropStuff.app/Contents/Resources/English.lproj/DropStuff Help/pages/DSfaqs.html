Frequently Asked Questions

Q: Does StuffIt use "lossy" or "lossless" compression?

A: StuffIt uses lossless compression.

Lossy compression is commonly used in multimedia files (ie: audio, image and movie files). It's a process whereby information is "intelligently" discarded to create a smaller file. There is a sacrifice however. Generally, the discarded information has a negligible affect on the file's image, sound or playback quality, but the process can be modified so that there is greater compression but lower quality. Because StuffIt can be used to store critical information such as spreadsheet or database files, it uses lossless compression, a process where files are made smaller without discarding any information. When a file is compressed and then expanded it is identical (bit for bit) to the original file - no data is lost.


Q: I am trying to compress files as small as possible. Any tips?

A: Some files compress better than others. Compression algorithms look for redundancy in files. Many multimedia files are already saved in a "lossy" compression format which reduces this redundancy. You may get some further compression on certain "lossy" compressed files (such as .gif images or .mp3 audio files) by using the SITx recompression options.

To do this go to DropStuff's preferences. In the StuffIt Preferences click on the Recompression tab. Check the boxes for each File Type that you are trying to compress.

Uncompressed multimedia formats such as .wav or .aiff (audio) and many high resolution graphics such as .psd (native PhotoShop picture), .eps (encapsulated postscript graphics), and uncompressed .tiff files will compress quite well. In fact, if you are starting out with a high resolution graphic you may get a smaller file with no loss of information by compressing into StuffIt X format instead of converting to a lossy format such as JPEG or GIF.

To further optimize your compression settings, try adjusting the Custom Compression Settings to see if this improves your results.


Q: I have a password encrypted file for which I have forgotten the password. What can be done?

A: There is nothing that we can do in this case: the contents of your file has been encrypted based on the password you provided, it cannot be decrypted without this password. There is no "backdoor", and we are not aware of any weakness that would allow you to break the encryption, short of trying all possible passwords. We apologize if this has caused you to lose an important file, but encryption would be a worthless feature if it could easily be bypassed.


Q: Who can expand StuffIt X (.sitx) archives?

A: Both Macintosh and Windows users can, provided they have a copy of our free expansion tool, StuffIt Expander. For the most backwards-compatible archive check the "Create Backwards Compatible Archives" option in DropStuff's StuffIt preferences. The most current version of StuffIt Expander can be downloaded from: StuffIt.com

Q: What happened to DropZip, DropTar, and DropSegment?

A: Starting with DropStuff version 9.0, DropStuff, DropTar, DropZip, and DropSegment have been combined together into a single drop box application, DropStuff. This allows more convenient access to features that were previously divided across multiple applications.
